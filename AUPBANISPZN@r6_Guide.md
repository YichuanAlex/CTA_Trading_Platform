#  策略业务说明文档

## 1. 策略概述
**截面调仓策略**。它旨在通过捕捉不同期货品种间的强弱关系，构建多头（做多强势品种）和空头（做空弱势品种）的投资组合，以获取超额收益。

策略的核心基于**主力连续合约**数据，确保在合约换月过程中交易信号的连续性和稳定性。

## 2. 核心业务逻辑

### 2.1 投资组合构建
策略不再局限于单一品种的涨跌，而是关注**品种间的相对表现**。
- **多头组合**：买入因子表现最强的一组品种。
- **空头组合**：卖出因子表现最弱的一组品种。
- **市场中性**：通过同时持有并通过动态调整多空仓位，策略在一定程度上对冲了市场整体的系统性风险。

### 2.2 核心因子模型
策略支持多种因子来衡量品种的“强弱”，可通过配置灵活切换：
*   **相对强弱因子**：基于收益率在中位数或特定分位数之上的表现，剔除市场平均波动的影响，捕捉更纯粹的超额动能。

### 2.3 主力连续合约机制
为了解决期货合约到期问题，策略内部集成了**主力合约自动切换**机制：
- **自动换月**：策略会自动跟踪持仓量或成交量最大的主力合约。
- **数据连续**：在计算动量等长期指标时，策略会自动处理合约切换带来的价格跳空，确保历史数据的平滑和因子的准确性。

## 3. 调仓与风控机制

### 3.1 详细调仓逻辑

策略的调仓过程是一个严谨的自动化流程，主要包含以下五个步骤：

1.  **触发检查 (Trigger)**
    *   **时间周期检查**：系统会计算距离上一次调仓的天数。只有当时间间隔达到预设的周期（如每N天、每周几）时，才会激活调仓流程。
    *   **交易日历过滤**：系统会自动跳过非交易日（如周末），顺延至下一个有效交易日进行检查，确保不会在休市期间触发无效信号。

2.  **前置校验 (Pre-check)**
    *   **交易时段保护**：为避免在市场流动性差或集合竞价时段交易，策略会校验当前时间是否在允许的交易窗口内（例如避开开盘前5分钟和收盘前几分钟）。如果不在交易窗口，系统会发出警告并推迟处理。
    *   **数据完整性校验**：检查所有关注品种的行情数据是否完整。如果发现某个品种缺乏最新数据（如行情中断），系统会发出警报，避免基于过时数据做出决策。
    *   **持仓一致性核对**：在执行前，策略会核对底层账户的实际持仓与策略记录的理论持仓是否一致，确保交易基于准确的持仓状态。

3.  **筛选与排序 (Selection & Ranking)**
    *   **因子计算与排序**：策略获取所有候选品种的最新因子值（如动量、价格等）。
        *   **做多逻辑**：按照因子值从高到低排序，选取排名靠前的品种作为多头候选。
        *   **做空逻辑**：按照因子值从低到高排序，选取排名靠后的品种作为空头候选。
    *   **持仓保留规则**：为了降低换手率，策略在筛选时会检查当前已持有的品种。如果现有持仓品种仍在合格名单内，倾向于保留，除非其排名大幅下滑跌出优选池。

4.  **资金分配 (Capital Allocation)**
    *   **等权分配**：根据设定的总初始资金和目标持仓数量，计算每个品种应分配的资金额度（例如：总资金 / 目标持仓数）。
    *   **手数取整**：将分配的资金除以当前品种的价格和合约乘数，并向下取整到最小交易单位（1手），得出具体的目标持仓手数。

5.  **交易执行 (Execution)**
    *   **差异比对**：系统将“目标持仓”与“当前持仓”进行逐一比对。
        *   **需平仓**：对于不在目标列表中、或目标数量减少的品种，执行卖出平仓操作。
        *   **需开仓/加仓**：对于新进入目标列表、或目标数量增加的品种，执行买入开仓操作。
    *   **反手处理**：如果某个品种从多头转为空头（或反之），系统会先执行平仓操作，确保净头寸归零后，再反向开仓，避免保证金不足或锁仓风险。
    *   **优胜劣汰清理**：对于那些因子表现变差、跌出优选名单的旧持仓，系统会生成专门的清理指令将其全部平仓，释放资金给新晋的强势品种。

### 3.2 资金管理与对冲
- **手动/自动干预**：支持自动运行，同时也保留了手动触发调仓的接口，便于在特殊市场环境下人工介入。
- **状态保存**：策略具备定期保存运行状态的功能，确保在系统重启或故障恢复后能接续之前的交易逻辑。

## 4. 关键配置说明

| 配置项名称 | 业务含义 |
| :--- | :--- |
| **因子类型选择** | 决定策略是依据“绝对价格”、“历史收益率”还是“相对分位数”来通过筛选品种。 |
| **计算窗口周期** | 设定计算动量或趋势指标时回溯的历史数据长度（如过去N个周期）。 |
| **分位阈值** | 当使用相对强弱因子时，用于定义“强势”和“弱势”的统计学标准。 |
| **交易品种池** | 设定策略关注和交易的期货品种列表（如铜、橡胶、螺纹钢等）。 |
| **调仓时间设定** | 规定策略在每周的星期几、什么时间点执行调仓检查。 |

## 5. 运行流程简述
1.  **初始化阶段**：系统启动时，首先回溯加载历史行情数据，预热计算各项因子指标。
2.  **实时监控**：盘中实时接收最新行情，持续更新主力合约状态和因子数值。
3.  **信号触发**：到达预设的调仓时间点，策略对比所有品种得分，生成目标持仓列表。
4.  **执行交易**：系统自动计算目标持仓与当前持仓的差异，发送相应的开平仓指令，完成调仓。
